Cycles vs. Nanoseconds Timing values are always in clock cycles in spec sheets. To convert to nanoseconds: ns = cycles / frequency(GHz). DDR4-3200 runs at 1600 MHz (1.6 GHz), so CL16 = 16 / 1.6 = 10 ns. Lower frequency RAM with the same CL number has higher absolute latency.

CAS Latency — CL

CAS Latency (CL), also written tCL or tCAS, is the delay in clock cycles between the READ command being issued and the first data bit appearing on the DQ pins. It represents the column access time from the open row buffer.

CL is the single most impactful timing for random-access latency. A row-buffer hit incurs only CL worth of delay. For sequential streaming, CL matters far less than bandwidth.

tRCD — RAS-to-CAS Delay

tRCD (Row Address to Column Address Delay) is the minimum number of cycles that must elapse between issuing an ACTIVATE command and issuing a READ or WRITE command to that bank.

This time allows the sense amplifiers to fully amplify and stabilize the row data. Issuing a column command too early, before sense amplification completes, would result in reading garbage or corrupting the cell data.

  • Physically: time for sense amp to reach full swing and the row buffer to settle
  • Typical DDR4: 13–17 ns absolute, regardless of frequency
  • A row buffer hit bypasses this delay entirely (no new ACT needed)

tRP — Row Precharge Time

tRP is the minimum cycles between a PRECHARGE command and a subsequent ACTIVATE on the same bank. It represents the time needed to equalize the bit lines back to VDD/2 after closing a row.

tRP and tRCD are often equal (both ~14–18 cycles on DDR4) because both are governed by the same sense amplifier circuit — one driving the lines up, one pulling them back to mid-rail.

tRAS — Active-to-Precharge Time

tRAS is the minimum cycles a bank must remain active (between ACT and PRE). It ensures that the sense amplifiers have had enough time to fully restore all cell charges in the open row before being disconnected.

tRAS Minimum tRAS ≥ tRCD + CL + BL/2 — the row must stay open long enough to finish a read burst before precharging. Violating this corrupts the cells that haven't been fully restored yet.

The Row Cycle Time (tRC) is the total time to open a row, use it, and open a different row in the same bank:

tRC = tRAS + tRP

Why tCCD_L vs tCCD_S matters

This is the key reason bank groups exist in DDR4/DDR5. A memory controller scheduling reads across different bank groups can issue them 4 cycles apart instead of 6+. Over millions of accesses per second, this difference is significant for throughput-sensitive workloads.

Tertiary Timings

Tertiary timings are rarely exposed in BIOS but matter for extreme overclocking and signal integrity:

Command Rate (CR)

The Command Rate (also called T1 or T2) defines how many clock cycles the memory controller waits between consecutive commands when switching between chip-selects (ranks or DIMMs).

  • 1T (CR1): Commands can be issued every clock — lowest latency, requires robust signal integrity
  • 2T (CR2): Commands issued every 2 clocks — easier for the controller and signal traces; adds 1 cycle of latency to every operation
  • 3T (Gear-3 in DDR5): DDR5 introduced Gear modes where the controller runs at a fraction of DRAM speed to ease SI at extreme frequencies

Bandwidth Formula

Peak theoretical memory bandwidth for a DRAM configuration:

Peak Bandwidth Calculation
BW (GB/s) = Transfer Rate (MT/s) × Bus Width (bytes) × Channels
Example: DDR5-6400, 64-bit bus, dual-channel:
BW = 6400 MT/s × 8 bytes × 2 = 102.4 GB/s
Example: DDR4-3200, 64-bit bus, dual-channel:
BW = 3200 MT/s × 8 bytes × 2 = 51.2 GB/s

Effective bandwidth is always lower than theoretical peak due to refresh overhead, timing gaps (tCCD, tRCD, tRP), read-to-write turnarounds, and command bus utilization.

XMP & EXPO — Overclocking Profiles

JEDEC defines conservative standard speeds (DDR4-3200, DDR5-4800) that work on all compliant hardware. Manufacturers create faster overclocking profiles stored in the SPD (Serial Presence Detect) EEPROM on the DIMM:

XMP — Extreme Memory Profile (Intel)

  • Developed by Intel for DDR3/DDR4/DDR5
  • Up to 3 profiles stored in SPD (XMP 3.0)
  • Enabled in BIOS/UEFI with one click
  • Profiles can include voltage, frequency, and all primary/secondary timings
  • XMP 3.0 adds user-writable custom profiles

EXPO — Extended Profiles for Overclocking (AMD)

  • AMD's alternative to XMP for Ryzen platforms
  • Introduced with DDR5 and Zen 4 (AM5)
  • Optimized for AMD's memory controller characteristics
  • Stored alongside XMP profiles in SPD
  • Tuned for AMD's Infinity Fabric (FCLK) relationship
Stability Note XMP/EXPO profiles are technically overclocks — they exceed JEDEC specifications. Instability can manifest as random crashes, memory errors under load, or POST failures. Always run a memory stability test (MemTest86, HCI Memtest) after enabling an XMP/EXPO profile.
