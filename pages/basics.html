What Is DRAM?

Dynamic Random-Access Memory (DRAM) is the primary form of working memory in virtually every modern computing device — from smartphones to supercomputers. When your operating system loads, when you open a browser, when a GPU renders a frame — the data lives in DRAM.

DRAM is random-access, meaning any memory location can be read or written in approximately the same amount of time, regardless of where it is in the array. This distinguishes it from sequential-access storage like tape or even some flash configurations.

DRAM is also volatile: it loses all stored data the moment power is removed. This is an inherent consequence of how it stores bits — in tiny capacitors that discharge over time.

Key Fact Modern DDR5 DIMMs can hold 64 GB on a single module and transfer data at over 51 GB/s — yet every bit is still stored using the same fundamental 1-transistor, 1-capacitor cell concept invented in 1968.

The 1T1C Cell

The fundamental storage element in DRAM is the 1T1C cell — one transistor and one capacitor. It is arguably the simplest possible memory cell, and that simplicity is why DRAM achieves such extraordinary storage density.

The Transistor (Access Switch)

The MOSFET transistor acts as a gating switch. Its gate is connected to the Word Line (WL). When the word line is driven high, the transistor turns on, connecting the storage capacitor to the Bit Line (BL). When the word line goes low, the transistor cuts off, isolating the capacitor.

The Capacitor (Storage Element)

The storage capacitor Cs holds charge to represent a binary value:

  • Logic 1 — capacitor is charged (holds approximately VDD/2 to VDD)
  • Logic 0 — capacitor is discharged (holds approximately 0 V)

Modern DRAM capacitors are extraordinarily small — on the order of 10–30 fF (femtofarads) — yet they must store enough charge to be reliably detected by sense amplifiers. This has driven decades of innovation in capacitor geometry: trench capacitors etched deep into the silicon substrate, and later stacked capacitors that grow upward above the cell.

Why "Dynamic"?

The word dynamic is the key differentiator. Unlike SRAM (which uses cross-coupled inverters to hold state indefinitely), a DRAM cell's capacitor leaks charge due to:

  • Subthreshold leakage through the off transistor
  • Junction leakage at the capacitor's diffusion contacts
  • Dielectric leakage through the capacitor oxide

A charged DRAM cell will completely discharge in roughly 32–64 milliseconds at room temperature (faster at elevated temperatures). The memory controller must therefore periodically refresh every cell — read it and write it back — before its charge decays below the detectable threshold.

Refresh Overhead Refresh operations consume memory bus bandwidth and add latency. For DDR4 at 85°C, the refresh interval (tREFI) is 3.9 µs — meaning the controller issues a refresh command every 3.9 µs. This isn't free: it reduces effective bandwidth by roughly 1–3%.

History & Evolution

DRAM has one of the longest and most consequential histories of any semiconductor component.

YearMilestone
1966Robert Dennard (IBM) invents the 1T1C DRAM cell concept
1970Intel 1103 — first commercially available DRAM chip (1 Kbit)
19734 Kbit DRAM; MOS process displaces core memory in computers
1980sDRAM density doubles roughly every 2–3 years; 64 Kbit → 4 Mbit
1993Samsung introduces synchronous DRAM (SDRAM) — clocked interface
2000DDR SDRAM — doubles bandwidth per clock via double data rate
2003DDR2 — reduced voltage (1.8 V), higher clock speeds
2007DDR3 — 1.5 V, up to 2133 MT/s
2012DDR4 — 1.2 V, up to 3200 MT/s, bank groups introduced
2015HBM1 — 3D-stacked DRAM on package (AMD Fury)
2021DDR5 — 1.1 V, 4800–8400+ MT/s, on-die ECC, 32 Gbit density
2022HBM3 — 819 GB/s per stack; LPDDR5X at 8533 MT/s
2024+HBM3E, LPDDR6, DDR5 at 12800 MT/s on the roadmap
Dennard Scaling Robert Dennard (same inventor as DRAM) also described "Dennard scaling" — the observation that as transistors shrink, power density stays constant. This scaling ended around 2006, which is why clock speeds plateaued and the industry shifted to multi-core processors and specialized memory like HBM.

Where DRAM Fits in the Memory Stack

Modern systems use a memory hierarchy because no single technology simultaneously offers the speed of SRAM and the density of Flash. DRAM occupies the critical middle tier:

Memory Hierarchy (latency increases downward)
CPU Registers — ~0.1 ns · bytes
L1 Cache (SRAM) — ~0.5 ns · 32–64 KB
L2 Cache (SRAM) — ~5 ns · 256 KB–4 MB
L3 Cache (SRAM) — ~30 ns · 8–64 MB
DRAM (Main Memory) — ~60–100 ns · GB–TB
NVMe SSD (Flash) — ~20–100 µs · TB
HDD — ~5–15 ms · TB
