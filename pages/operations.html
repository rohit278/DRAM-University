The fundamental rule: a bank must be precharged (idle) before it can be activated, and activated before it can be read or written.

PRECHARGE (PRE)

Before a new row can be opened, the bit lines must return to their equilibrium voltage (VDD/2). This is called precharging. The PRECHARGE command closes the currently active row and initiates bit-line equalization.

What happens electrically

  1. The word line for the active row is de-asserted (transistors turn off)
  2. An equalization circuit connects BL and BL̄ together through a resistor, driving both to VDD/2
  3. After time tRP (Row Precharge time), the bank is ready for a new ACTIVATE
Precharge All (PREA) The PRECHARGE ALL command precharges every bank simultaneously in one command. Useful before refresh or self-refresh entry.

ACTIVATE (ACT)

The ACTIVATE command opens a row in a bank. It is issued with a row address and bank address. The memory controller must wait at least tRAS (minimum active time) before precharging, and at least tRCD (RAS-to-CAS delay) before issuing a column command.

What happens electrically

  1. The row address decoder drives the selected word line high
  2. All access transistors in that row turn on — connecting every cell to its bit line
  3. Each cell's capacitor shares charge with its bit line, creating a tiny differential voltage (±50–100 mV)
  4. Sense amplifiers detect and amplify this differential to full logic swing (VDD / GND)
  5. The amplified values are written back to the capacitors (restoring destructively-read data)
  6. The entire row now sits in the row buffer — ready for fast column access

READ

After ACTIVATE, the memory controller issues a READ command with a column address. The DRAM selects the specified columns from the row buffer and drives the data onto the DQ pins.

Burst Length

DRAM always transfers data in bursts — a contiguous block of columns starting at the given address. Burst Length (BL) is configurable:

  • BL4: 4 transfers (used in some LPDDR and graphics modes)
  • BL8: 8 transfers (standard for DDR3/DDR4)
  • BL16: 16 transfers (DDR5)

At DDR4-3200, a BL8 burst transfers 8 × 64 bits = 64 bytes (one CPU cache line) in 4 clock cycles (8 transfers at 2 per clock).

Auto Precharge

A READ with the AP bit set automatically precharges the bank after the burst completes. This is useful for random-access patterns where the next access will be to a different row — it eliminates the need for a separate PRE command.

WRITE

The WRITE command works similarly to READ but in reverse: column-addressed cells in the row buffer are overwritten with the data driven on the DQ pins by the controller. The written values are also stored back to the capacitors as long as the row remains active.

Write Latency Unlike reads (which have CAS latency before data appears), writes have a write latency (WL) that is typically CL − 1 for DDR3 and programmable for DDR4/DDR5. Data must be driven exactly at the right time relative to the WRITE command.

REFRESH (REF)

Because DRAM cells lose charge over time (see Module 01), the memory controller must periodically issue REFRESH commands to restore all cells before they decay below the readable threshold.

Refresh Mechanics

  • A REF command causes the DRAM to internally activate and precharge one or more rows
  • The DRAM's internal refresh counter tracks which rows need refreshing next
  • All banks must be precharged before a REF command is accepted
  • During refresh, the DRAM is unavailable for reads/writes — this is the refresh penalty

Refresh Modes

  • Auto-Refresh (AR): Controller issues REF commands at tREFI intervals; DRAM manages row selection internally
  • Self-Refresh (SR): DRAM manages its own refresh internally — used during power-down (e.g., DRAM sleep on phones). The controller asserts CKE low to enter self-refresh
  • Same Bank Refresh (SBR, DDR5): Refreshes one bank at a time, leaving others accessible — dramatically reduces refresh impact on latency

Command Encoding

DRAM commands are encoded on the CA (Command/Address) bus using combinations of the RAS#, CAS#, WE#, and CS# pins. In DDR4 and DDR5, the CA bus carries multiple bits per clock cycle.

DDR5 Command Changes DDR5 uses a 14-bit CA bus clocked at the full data rate, and many commands have been reorganized. The separate RAS#/CAS#/WE# encoding is replaced by a unified command field, making the protocol more extensible for future features.
