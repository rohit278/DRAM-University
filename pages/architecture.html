Module 02 — Architecture

DRAM Architecture

How billions of 1T1C cells are organized into arrays, subarrays, banks, ranks, and channels — and how the memory controller orchestrates access to all of it.

Cell Array

Individual 1T1C cells are arranged into a two-dimensional cell array — a grid of rows and columns. Each row is connected by a Word Line (WL) and each column by a pair of Bit Lines (BL / BL̄) in a differential configuration.

A typical DRAM array might have 32,768 rows × 1,024 columns. Activating a single word line exposes an entire row of data — up to 8 KB — to the sense amplifiers simultaneously. This row is called the open row or active row.

Sense Amplifiers (SA)

Sense Amplifiers

When a word line is activated, the tiny charge from each cell flows onto its bit line, causing a minuscule voltage change — typically just 50–100 mV. A sense amplifier (SA) sits at the end of each bit line pair and detects this differential voltage, then amplifies it to full VDD/GND swing.

How a Sense Amplifier Works

A standard DRAM sense amplifier is a cross-coupled CMOS latch:

  1. Precharge phase: Both BL and BL̄ are precharged to VDD/2 (equalized)
  2. Access phase: WL activates; the cell's capacitor shares charge with BL. One side rises slightly, the other falls
  3. Sense phase: The SA is enabled. The latch regenerates: the higher side is driven to VDD, the lower to GND
  4. Restore phase: The amplified voltage on BL writes back to the capacitor, restoring its charge (destructive read!)
Destructive Read DRAM reads are destructive — activating a row partially discharges every cell in that row. The sense amplifier restore step is mandatory. If power is cut mid-sense, data is lost.

Subarrays & Mats

A full DRAM array with 32K rows and 1K columns would have extremely long bit lines. Long bit lines have high capacitance, which slows sensing and increases power. Instead, modern DRAM divides the array into subarrays (also called mats), each with its own local sense amplifiers.

A typical DRAM chip might have 128–512 subarrays per bank, each with ~512 rows and ~512 columns. Subarrays in the same bank share row and column decoders but have independent sense amplifiers.

Banks

A bank is an independently operable section of DRAM. Each bank has its own:

  • Row address decoder
  • Column address decoder
  • Sense amplifier array (row buffer)
  • Precharge and refresh logic

Banks share the external data bus (DQ pins) but can operate their internal pipelines concurrently. This parallelism is exploited by memory controllers to hide latency: while one bank is precharging, another can be activated and read.

Bank Groups (DDR4/DDR5)

DDR4 introduced bank groups: banks are clustered into groups (typically 2 or 4), each with its own data path and I/O circuitry. A back-to-back CAS command to different bank groups can be issued with much shorter delay (tCCD_S) than to the same bank group (tCCD_L). This enables the controller to sustain very high throughput by interleaving across bank groups.

Ranks & Channels

Ranks

A rank is a set of DRAM chips that are selected together and respond simultaneously to a single command. All chips in a rank share the same chip-select (CS) signal. On a typical DDR4 DIMM with x8 chips and a 64-bit bus, you need 8 chips to form one rank (8 chips × 8 bits = 64 bits).

A dual-rank DIMM has two independent sets of chips that alternate access. The memory controller interleaves between ranks to hide precharge latency.

Channels

A memory channel is a complete, independent electrical interface between the CPU's memory controller and one or more DIMMs. Each channel has its own:

  • 64-bit data bus (DQ[63:0])
  • Command/address bus
  • Clock signals

Modern desktop CPUs have 2 channels; high-end desktop (HEDT) and server CPUs may have 4, 6, 8, or even 12 channels. Each additional channel multiplies peak memory bandwidth proportionally.

Address Mapping

A physical memory address issued by the CPU must be decoded into: channel → rank → bank group → bank → row → column. The exact mapping varies by memory controller implementation and can be tuned for performance.

Row interleaving (spreading consecutive addresses across different banks) maximizes bank-level parallelism. Cache-line interleaving keeps a CPU cache line within one bank, which can improve row-buffer hit rates for streaming workloads.

Row Buffer Policy Memory controllers choose between open-page (leave row active for future hits), closed-page (precharge immediately after access), and adaptive policies. Open-page wins for streaming; closed-page wins for random access.

Memory Controller

The memory controller (integrated into the CPU die on all modern processors since ~2010) is responsible for:

  • Accepting read/write requests from the CPU cache hierarchy
  • Translating physical addresses to DRAM coordinates
  • Scheduling DRAM commands (ACT, CAS, PRE) to maximize bandwidth and minimize latency
  • Issuing periodic refresh commands
  • Managing power-down states during idle periods
  • Implementing ECC error detection and correction (on ECC systems)

Command Scheduling

Modern controllers use sophisticated out-of-order scheduling: incoming requests are reordered to exploit row-buffer hits, bank parallelism, and bus turnaround efficiency. A simple FCFS (First-Come, First-Served) scheduler leaves substantial bandwidth on the table.
